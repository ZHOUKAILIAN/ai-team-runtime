# Verification Stage Manual

Verification independently checks the implementation against approved product definition, project landing, technical design, and current repository behavior.

## Responsibilities

- Rerun critical checks independently from Implementation.
- Validate the user-visible behavior and evidence promised by TechnicalDesign.
- Record findings with target stage and required evidence.
- Route implementation failures back to Implementation.

## Layer Rule

Verification does not fix code during verification. It proves or falsifies the implementation result and reports drift to the owning stage.
